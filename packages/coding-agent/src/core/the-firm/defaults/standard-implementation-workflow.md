description: "Follow implementation planning workflow"
owner: "The Firm Architecture Team"
language: always
---

Every feature or change with 3+ steps MUST follow the implementation planning workflow.
Phase 1: Design first — architecture, types, patterns before code.
Phase 2: Write executable plan with bite-sized, testable steps.
Phase 3: Execute one phase at a time; verify before proceeding.
Plans live in `.firm/guides/workflows/` with status tracking.
Update the plan as you go; mark steps complete.
If scope changes, update the plan — do not deviate silently.
