# The Firm Documentation

This directory contains the public-facing documentation for The Firm.

## Documents

- `overview.md` — what The Firm is and what problem it solves
- `how-it-works.md` — the operating model in practical terms
- `getting-started.md` — how to think about using The Firm in greenfield and brownfield projects
- `bootstrap.md` — how to scaffold a repo under The Firm
- `architecture.md` — system structure, components, and data flow
- `governance.md` — councils, escalation rules, and decision authority
- `source-of-truth.md` — what The Firm treats as canonical and why
- `workspace-policy.md` — explicit workspace binding and path discipline
- `brownfield-adoption.md` — detailed adoption path for existing repositories
- `runtime-agents.md` — the role structure of the live runtime agent bench
- `issue-workflow.md` — how Git, Beads, and `.firm/` artifacts work together
