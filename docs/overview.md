# Overview

The Firm is a software delivery operating system for AI-assisted work.

It is designed for teams or individuals who want more than a generic coding assistant. Instead of relying on one broad agent with loose tasking, The Firm organizes work like a disciplined software firm.

## The core model

The Firm combines:

- client intake and engagement handling
- issue-first execution
- specialist roles
- artifact-based handoffs
- independent QA
- risk-tiered verification
- explicit governance and escalation

## Why it exists

AI coding systems often fail in predictable ways:

- they optimize for speed instead of the right outcome
- they blur planning, implementation, and QA into one role
- they lose context over long-running work
- they claim completion before proof exists
- they become hard to control in brownfield codebases

The Firm exists to solve those problems with operating discipline rather than prompt tricks.

## What it runs on

The Firm is designed to run on top of Pi.

- Pi provides the runtime and tools
- Beads provides the issue-control plane
- Dolt provides versioned workflow truth
- The Firm provides doctrine, workflow, governance, and runtime role structure
